# Day 17 — GitHub Polish and Documentation

## Goal

Polish the portfolio projects so they look professional on GitHub and are easy to understand for a recruiter, analyst, or technical interviewer.

## What I worked on

- Cleaned project folders and file naming
- Checked that scripts and apps run from a clean session
- Wrote or improved README files
- Added project descriptions, business logic, and how-to-run instructions
- Prepared screenshots for GitHub
- Made the projects easier to present in an interview

## What I need to remember

A project is not only judged by whether the code works.

It is also judged by:

- whether the business problem is clear (dont forget why we do this)
- whether the folder structure is clean
- whether the code can be run by someone else
- whether the README explains the workflow
- whether the outputs are easy to understand
- whether I can explain the logic simply

## Interview explanation

These projects show practical R skills for insurance analytics:

- data cleaning
- reporting
- Shiny dashboards

- structured outputs
