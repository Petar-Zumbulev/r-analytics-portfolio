# Day 18 — Interview Prep

## Goal

Prepare to explain my R insurance analytics portfolio project clearly and solve realistic R interview-style tasks.

## What I need to be ready for

- explain my Shiny dashboard project
- explain the PDF/OCR/API workflow project
- write clean dplyr code under pressure
- explain insurance metrics such as claim count, severity, premium, loss ratio, and inflation
- explain why the project structure is professional
- explain how Git, Excel export, and reporting fit into the workflow

## Main interview message

I built this project to simulate a practical analyst workflow inside an insurance or reinsurance company. The goal was not only to calculate metrics, but to create a clean, reproducible R workflow that takes raw claims-style data, cleans it, calculates business KPIs, visualizes trends, and presents the results in a Shiny dashboard with exportable outputs.

## Key skills shown

- R project structure
- tidyverse data cleaning
- dplyr summaries
- ggplot2 visualization
- Shiny dashboard development
- Excel export
- reusable functions
- insurance KPI logic
- GitHub documentation
- reporting mindset

## What I should emphasize

This project is not just a dashboard. It is a full reporting workflow:

raw-style data → cleaning → validation → metrics → plots → Shiny app → Excel export → README documentation
